# SECURITY.md — Архитектура безопасности LexAI

## Модель угроз

| Угроза | Статус | Решение |
|--------|--------|---------|
| Утечка API-ключа | ✅ Решено | Ключ только на бэкенде, никогда не в клиенте |
| Хранение документов | ✅ Решено | Документы не сохраняются нигде |
| Перехват в transit | ✅ Решено | HTTPS everywhere, TLS 1.3 |
| Взлом устройства пользователя | ⚠️ Вне scope | Документ читается в RAM браузера, не пишется на диск |
| Взлом сервера LexAI | ✅ Решено | На сервере нет документов — нечего красть |
| Утечка от Anthropic | ⚠️ Политика | Anthropic не хранит и не обучается на API-данных |

---

## Архитектура: почему нельзя "полностью локально"

Claude — это облачная модель. Полностью локальный анализ без передачи данных возможен
только с open-source моделями (Llama, Mistral), но качество анализа юридических документов
будет значительно ниже. Это компромисс между безопасностью и качеством.

**Решение:** бэкенд-прокси. Документ уходит только до нашего сервера → сервер → Anthropic.
На нашем сервере документ не сохраняется.

---

## Схема передачи данных

```
Устройство пользователя
    │
    │  HTTPS (TLS 1.3)  — только текст документа
    ▼
Сервер LexAI (FastAPI)
    │   - Проверяет JWT токен пользователя
    │   - НЕ сохраняет текст документа
    │   - Добавляет API-ключ Anthropic
    │
    │  HTTPS — текст документа + system prompt
    ▼
Серверы Anthropic (США)
    │   - Анализирует документ
    │   - НЕ сохраняет для обучения (API policy)
    │
    │  Ответ (JSON с анализом)
    ▼
Сервер LexAI
    │   - Возвращает анализ пользователю
    │   - НЕ кеширует документ
    ▼
Устройство пользователя
    - Результат отображается в браузере
    - Нигде не сохраняется (только в RAM)
```

## Что хранится на сервере LexAI

| Данные | Хранится? | Где |
|--------|-----------|-----|
| Текст документа | ❌ Нет | — |
| Результат анализа | ❌ Нет (по умолчанию) | — |
| История анализов (Pro) | ✅ Опционально | Зашифрованная БД, только хеш |
| JWT токен пользователя | ✅ Да | Redis, TTL 24h |
| Email / аккаунт | ✅ Да | PostgreSQL |
| API-ключ Anthropic | ✅ Да | env variable, не в БД |

## Рекомендации по инфраструктуре

### Сервер
- VPS в нужной юрисдикции (для EU клиентов — ЕС-датацентр, для RU — РФ)
- Firewall: только 443 (HTTPS) открыт наружу
- Fail2ban против брутфорса
- Автообновление системы (unattended-upgrades)

### Бэкенд (FastAPI)
- Rate limiting: max 10 req/min на пользователя
- Максимальный размер документа: 100KB (защита от злоупотреблений)
- Логи: только метаданные (user_id, timestamp, jurisdiction) — без текста документа
- API-ключ в env, никогда в коде или БД

### БД (PostgreSQL)
- Шифрование at-rest (pg_crypto или managed DB с шифрованием)
- Резервные копии зашифрованы
- Пароль БД не в коде — только в env

### GDPR / ФЗ-152
- Пользователь явно соглашается (ConsentModal) перед первым анализом
- Право на удаление аккаунта и всех данных
- DPA с Anthropic (они предоставляют для API-клиентов)
- Для EU: сервер в ЕС или SCCs с Anthropic

## Что делать при инциденте

1. Немедленно ротировать API-ключ Anthropic
2. Инвалидировать все JWT токены (flush Redis)
3. Уведомить пользователей в течение 72 часов (требование GDPR)
4. Сформировать отчёт об инциденте

---

## Вывод

Главный актив безопасности этого приложения: **на сервере нечего красть**.
Документы не хранятся. Даже при полном компрометировании сервера
злоумышленник получит только список email-адресов и метаданные запросов.
